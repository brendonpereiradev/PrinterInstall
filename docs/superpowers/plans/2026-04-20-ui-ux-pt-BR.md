# UI/UX pt-BR (WPF) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Entregar interface fixa em português do Brasil, localização híbrida (XAML + `.resx`), estados da grade em pt-BR, mensagens de validação e fluxos controlados em pt-BR, log técnico misto (pt-BR para marcos + texto remoto/EN quando aplicável), refinamentos visuais admin-neutros e layout principal opção A, sem alterar mensagens dentro de `PrinterInstall.Core`.

**Architecture:** Dicionários de recursos XAML fundidos em `App.xaml` para textos declarados nas views; `Resources/UiStrings.resx` com gerador para strings usadas em ViewModels; classe estática `TargetMachineStateDisplay` no App mapeia `TargetMachineState` → rótulo pt-BR para coluna e para prefixos de log; `TargetRowViewModel` expõe `StateDisplay` atualizado com `State`. Cultura de UI fixa `pt-BR` definida no arranque da aplicação. Nenhum arquivo de idioma na Core.

**Tech stack:** .NET 8, WPF, CommunityToolkit.Mvvm, `xunit` + `Microsoft.NET.Test.Sdk` no novo projeto de testes do App.
