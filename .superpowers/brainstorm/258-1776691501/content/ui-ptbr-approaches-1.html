Como implementar UI em pt-BR no WPF?

App só em português do Brasil — três caminhos típicos. Clique na opção que mais combina com o que você quer manter no código.

A

Dicionários XAML de recursos

Strings como DynamicResource / StaticResource em arquivos .xaml (ex.: Strings.pt-BR.xaml). Um dicionário “ativo” carregado no App.xaml. Bom para labels em XAML; ViewModels podem receber textos via serviço ou chaves.

B

Arquivos .resx (satélite)

Padrão .NET: Resources.resx + Resources.pt-BR.resx gera assembly satélite. Uso em código C# com Resources.NomeDaString ou extensão de markup. Forte para mensagens geradas no Core / ViewModels.

C

Híbrido

XAML com dicionários para títulos, botões e rótulos; .resx (ou um pequeno catálogo de strings em C#) para logs, erros e textos montados em código no PrinterInstall.Core. Reduz duplicação e mantém parsing/invariant culture onde já existe.

Recomendação rápida

  • Para um app WPF já com muito binding em XAML, C (híbrido) costuma equilibrar bem tela vs. mensagens de operação remota.

Observação

  • Como o idioma é fixo, dá para simplificar: um único dicionário/recurso pt-BR sem cultura dinâmica, se preferir menos infraestrutura.
